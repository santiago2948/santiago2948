## 💼 Experience

**Insurance Company — Backend Developer** *(recently completed)* · `Custom Python` · `React`

Worked under **strict constraints on external libraries**, which pushed me to design custom solutions from the ground up:

- Built a **policy management & visualization system** — architected the backend and data layer for managing and displaying insurance policies with interactive dashboards.
- Implemented **JWT-based authentication** and **in-house Python libraries** when third-party frameworks were off-limits — including a custom **"mini Flask"** HTTP layer built from scratch to handle routing, request/response flow, and API structure without relying on Flask itself.
- Designed a **hybrid bot + human chat system** in **n8n**, integrated with the web app via **React polling** — automatically responding to customers and **routing escalated bot cases to the right human agent**, eliminating the backlog of unassigned client cases.

**Tigo Colombia — Internship** *(Data & Operations)* · `Django` · `Pandas` · `scikit-learn`

- Ran **exploratory data analysis (EDA)** ahead of the dashboard — heavy use of **Pandas**, **NumPy**, **Matplotlib**, and **scikit-learn** (**Random Forest**) to understand visit patterns and inform what the tool should surface.
- Developed a **real-time metrics dashboard** for **same-day technical visits**, fed by a **daily data pipeline** that extracts operational data to keep insights up to date.
- Built for **network analysts** to spot **mass-failure patterns** across infrastructure — e.g. detecting when 50 customer outages stem from a single **node failure** instead of 50 isolated issues.
- Enabled **cost-saving operational decisions**: open one case at the node level and resolve all downstream faults in sequence, rather than dispatching dozens of redundant field visits to the same area.

**Easy Logistics — Internship** *(International Logistics)* · `Flask`

- Built a **container tracking module** with **third-party API integration** to locate shipments in **real time** — giving clients a more complete, transparent experience.
- Displayed the **full status history** of each container, the **planned route ahead**, and **current progress** on an **interactive map**.
- Strengthened the **client–company connection** by surfacing end-to-end shipment visibility instead of isolated status updates.

**AFACOM — Social Service** *(Scholarship Program)* · `Express.js` · `MongoDB` · `Vue.js`

- Developed the backend for a platform serving **AFACOM** — an association of **Social Communication faculties** across Colombia and their **research groups**.
- Built **Express.js** APIs to expose research groups, their **research projects**, and **KPIs**; stored and queried data in **MongoDB**.
- Contributed to the **Vue.js** frontend with **TypeScript**, supporting the UI that presents group profiles, research output, and performance metrics.

## 🏆 Featured Project — El Que Sabe

**Service marketplace** built for **Proyecto Integrador 1** — a university integrator course — based on a challenge from [**Magneto Global**](https://www.magneto365.com/) *(recruitment platform connecting freelancers and contractors)*.

The premise: a **services marketplace** where users request jobs like plumbing and find professionals with the right **skills**, **reviews**, **comments**, and a **work portfolio** — hiring through service provision rather than traditional employment.

**Architecture:** decoupled **React 18** client ↔ **Express.js** REST API ↔ **MySQL** — search freelancers by city/keyword, client & freelancer signup, auth, contract posts, and profile management.
